# 仓库维护

这份文档记录当前仓库已经补上的轻量维护能力，方便后续继续整理时直接复用。

## 当前已有

- `.editorconfig`：统一 UTF-8、LF、文件末尾换行
- `.gitattributes`：减少不同系统下的换行噪音
- `scripts/check-markdown-links.ps1`：检查仓库内 Markdown 相对链接
- `.github/workflows/markdown-links.yml`：在 `push` / `pull request` 时自动检查本地相对链接

## 本地检查命令

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\check-markdown-links.ps1
```

## 维护建议

- 优先补导航、补说明、补断链，不急着大规模改名
- `readme.md` 尽量只做入口页，不要继续堆长正文
- 图片和资源文件优先就近存放，避免根目录继续变杂
- 新增长文时，优先独立成文件，再从入口页回链

## 建议维护清单

- 新增或移动文档后，检查对应目录的 `readme.md`
- 新增图片后，确认是否应该放在更靠近主题的位置
- 修改目录职责后，同步更新根 [readme.md](../readme.md) 和 [仓库地图](./仓库地图.md)
- 提交前运行一次本地链接检查脚本

## 建议频率

- 日常补笔记后：至少检查一次当前目录入口
- 一次性整理多个目录后：运行一次完整仓库的链接检查
- 目录结构发生变化时：同步回看 [文档整理约定](./文档整理约定.md)

# 仓库地图

这份文档用于说明仓库目前的内容分层，以及后续继续整理时建议遵循的规则。

## 仓库定位

这是一个偏个人化的学习知识库，内容跨度较大，但主题相对清晰，主要集中在以下几条主线：

- SLAM / 机器人方向学习
- 算法刷题与求职准备
- 计算机基础知识整理
- Linux / Python 实用笔记
- 少量生活记录

当前最合适的整理方式不是大规模搬迁文件，而是：

- 先把一级目录做成稳定入口
- 再用各目录 `readme.md` 做索引
- 最后逐步补充子目录说明和命名规范

## 推荐整理规则

### 1. 一级目录尽量稳定

新增内容前先判断能否归入已有主题，尽量不要增加新的一级目录。

### 2. `readme.md` 只做导航

一个目录下的 `readme.md` 最好只承担三件事：

- 说明这个目录是干什么的
- 给出推荐阅读顺序
- 链接到具体笔记

正文内容尽量写到独立 Markdown 文件里。

### 3. 图片优先“就近存放”

- 某篇笔记专用图片：放在该主题目录内
- 多处会复用的图片：放根目录 `images/`

### 4. 命名尽量语义化

优先使用能直接看懂主题的名字，例如：

- `ros学习笔记.md`
- `体重管理.md`
- `动态规划.md`

尽量少用只有作者自己知道含义的缩写。

### 5. 外链放在目录入口集中管理

像教程链接、参考博客、资料站点，适合集中写在对应目录的 `readme.md` 中，而不是散落在根 README。

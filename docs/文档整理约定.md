# 文档整理约定

这份文档用于记录当前仓库整理时采用的一些轻量规则，方便后续继续补内容时保持风格一致。

## 1. 根目录只做总导航

- 根 `readme.md` 负责说明仓库定位和一级目录用途
- 不再把大量外链、公式示例、杂项备忘直接堆在首页

## 2. 一级目录优先维护 `readme.md`

一个主题目录的 `readme.md` 建议只做三件事：

- 说明这个目录是干什么的
- 给出推荐阅读顺序
- 链接到具体笔记或子目录

## 3. 正文笔记尽量独立

- 长内容尽量写进独立 Markdown 文件
- `readme.md` 更适合作为入口页，而不是正文全集
- 历史上已经写成“入口兼正文”的文件，可以先保留，再在顶部补导读说明

## 4. 标题层级尽量稳定

建议优先采用：

- `# 文件主题`
- `## 目录 / 推荐阅读 / 使用说明`
- `## 正文章节`

对于历史文件，如果全文已经大量使用 `# 第一章 / # 第二章` 这类结构，不必一次性全部重写，可以先在顶部补一段说明。

## 5. 占位页要写明状态

如果某篇文件当前内容还没展开，不要只留一个标题，建议明确说明：

- 当前是占位入口
- 后续准备补什么
- 应该先从哪里看起

## 6. 图片尽量就近存放

- 专题专属图片：放到对应专题目录
- 图片优先放到对应专题目录，尽量就近维护
- 迁移图片时优先保证现有 Markdown 链接不失效

## 7. 链接优先使用本地相对路径

- 仓库内部文档优先相对路径
- 外部资料再使用网页链接
- 修文档时顺手检查明显的断链和坏图

## 8. 整理策略以“低风险”为主

优先做这些事情：

- 补导航
- 补目录说明
- 修断链
- 标记缺失资源

尽量少做这些事情：

- 大规模改名
- 大批量移动文件
- 为了形式统一而重写旧笔记正文

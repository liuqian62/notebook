# 动态 SLAM 与语义 SLAM

这页想解决两个经常混在一起的问题：

1. 动态 SLAM 到底在解决什么
2. 语义 SLAM 又比普通 SLAM 多了什么

最短答案是：

- 动态 SLAM 主要解决“场景里有人和车在动，别把它们当静态地图的一部分”
- 语义 SLAM 主要解决“机器人不只知道几何，还要知道这是什么东西”

它们经常一起出现，但并不是同一个问题。

## 1. 为什么传统 SLAM 在真实场景里会出问题

很多经典视觉 SLAM 默认有一个重要假设：

- 场景大部分是静态的

这个假设在实验室、空房间、规则数据集上通常问题不大，但一到真实环境就会遇到：

- 行人走动
- 车流
- 门开关
- 显示器画面变化
- 机械臂、推车、宠物等局部动态物体

如果系统把这些动态区域也当成稳定约束，就可能出现：

- 特征跟踪被污染
- 位姿估计漂移
- 地图里留下“幽灵”
- 回环和重定位变差

## 2. 动态 SLAM 主要在做什么

动态 SLAM 的核心目标通常不是“理解物体”，而是先保证：

- 运动中的东西不要破坏定位建图

常见策略可以粗分为三类：

### 2.1 先检测动态区域，再剔除

这是最直接的一类做法。

思路是：

- 找出可能在动的人、车、椅子、门等区域
- 不让这些区域参与位姿估计或建图

优点：

- 容易接在现有 SLAM 系统上
- 工程改造路径清晰

缺点：

- 检测错了就会误杀静态信息或漏掉动态干扰

### 2.2 利用多视图几何判断运动一致性

这类方法不一定需要深度网络，而是通过：

- 极线几何
- 光流一致性
- RANSAC 残差
- 多视图三角化一致性

来区分：

- 跟随相机运动的静态背景
- 独立运动的前景物体

优点：

- 不依赖语义类别
- 能发现“本来可移动但此刻真的在动”的区域

缺点：

- 当相机本身运动复杂时，判别会更难
- 纯几何方法对遮挡、纹理差和噪声比较敏感

### 2.3 联合建模静态背景和动态物体

这是更进一步的路线。

它不只是“删掉动态物体”，而是显式估计：

- 相机轨迹
- 静态地图
- 动态物体轨迹或状态

这种做法更强，但代价也更高。

## 3. 语义 SLAM 主要在做什么

语义 SLAM 的重点不是只估计几何，而是让地图带有“可理解的标签”。

例如系统不只知道这里有一片表面，还知道：

- 这是墙
- 那是地面
- 这里有桌子
- 前面是椅子或人

于是地图从“几何地图”变成了：

- 几何 + 类别 + 有时还有实例

## 4. 语义信息为什么有价值

因为很多机器人任务并不只需要位姿，还需要知道：

- 哪些区域可通行
- 哪些物体可交互
- 哪些是动态高风险目标
- 哪些地方是门、楼梯、桌面、把手

语义 SLAM 常见价值包括：

- 更好的场景理解
- 更强的人机交互能力
- 为任务规划提供高层信息
- 让地图更接近“可用的世界模型”

## 5. 动态 SLAM 和语义 SLAM 的关系

两者常常一起出现，因为语义能帮助动态判断。

比如：

- 如果检测到“人”或“车”，它们更可能是动态目标

但它们并不等价。

### 5.1 动态 SLAM 不一定需要语义

纯几何方法也可以做动态剔除，比如：

- 运动一致性检测
- 光流异常
- 多视图残差

### 5.2 语义 SLAM 也不一定专门处理动态

有些语义 SLAM 的重点是：

- 让地图带上类别标签
- 输出语义 mesh 或语义体素图

它未必专门估计动态物体运动。

## 6. 一个很实用的理解框架

可以把三种系统简单区分成：

### 6.1 经典几何 SLAM

目标：

- 稳定定位
- 静态场景建图

地图：

- 稀疏点、稠密几何、关键帧图等

### 6.2 动态 SLAM

目标：

- 在动态场景中保持鲁棒定位
- 尽量构建静态背景地图

地图：

- 静态地图为主
- 有时附加动态目标轨迹

### 6.3 语义 SLAM

目标：

- 在地图中注入语义信息
- 让机器人不仅“定位”，还能“理解”

地图：

- 语义点云
- 语义体素
- 语义 mesh
- 语义实例图

## 7. 动态 SLAM 常见技术路线

### 7.1 几何剔除路线

典型思路：

- 先跑特征匹配 / 光流
- 用极线约束、基础矩阵、PnP 残差找运动不一致点
- 剔除这些点后再估计位姿

适合：

- 不想重度依赖语义网络
- 想保留传统系统结构

### 7.2 语义分割 / 实例分割辅助路线

典型思路：

- 用语义分割或实例分割先找到人、车、动物等潜在动态类别
- 再结合几何一致性做二次筛选

这类方法很常见，因为单靠语义类别也有问题：

- 人有时是静止的
- 椅子通常静止，但也可能被搬动

所以更稳的做法往往是：

- 语义给先验
- 几何做验证

### 7.3 联合优化路线

更进一步的方法会显式建模：

- 相机状态
- 静态点
- 动态目标轨迹
- 目标包围盒或物体级状态

优点：

- 信息利用更完整

缺点：

- 模型复杂
- 实时性压力大

## 8. 语义 SLAM 常见技术路线

### 8.1 2D 语义投影到 3D 地图

这是最常见、也最工程化的一条路：

1. 在图像上做语义分割或实例分割
2. 利用深度 / 三角化 / 点云，把标签投影到 3D
3. 在 3D 地图中累计和融合标签

优点：

- 实现路径清晰
- 容易和 RGB-D / stereo / LiDAR 系统结合

### 8.2 语义体素 / 语义 TSDF / 语义 OctoMap

这类方法重点是：

- 在体素层维护类别概率或语义标签

适合：

- 三维重建
- 规划
- 长期地图维护

### 8.3 语义 mesh / 场景图

更高层的做法会输出：

- 带语义标签的 mesh
- 物体关系图
- 场所-物体-人 的层次表示

这类地图更接近：

- Spatial AI
- 世界模型

## 9. 代表系统可以怎么理解

### 9.1 DynaSLAM

`DynaSLAM` 是动态视觉 SLAM 的经典代表之一。项目页明确说明它建立在 `ORB-SLAM2` 之上，增加了 dynamic object detection 和 background inpainting，能够在 monocular、stereo、RGB-D 动态场景中运行  
来源：<https://bertabescos.github.io/DynaSLAM/>

它很适合理解的一点是：

- 动态 SLAM 不一定从头重做一套系统
- 也可以是在经典 SLAM 上增加动态检测与静态背景恢复

### 9.2 DS-SLAM

`DS-SLAM` 的论文摘要说明它是一个面向动态环境的 semantic visual SLAM，系统中包含 tracking、semantic segmentation、local mapping、loop closing 和 dense semantic map creation 五个并行线程  
来源：<https://www.catalyzex.com/paper/ds-slam-a-semantic-visual-slam-towards>

它很适合用来说明：

- 动态处理和语义建图是可以在同一系统里同时做的

### 9.3 MaskFusion

`MaskFusion` 的摘要把它描述为 real-time、object-aware、semantic and dynamic RGB-D SLAM system，能够识别、分割、跟踪并重建多个独立运动物体  
来源：<https://www.researchgate.net/publication/324769414_MaskFusion_Real-Time_Recognition_Tracking_and_Reconstruction_of_Multiple_Moving_Objects>

它代表的方向是：

- 不只是剔除动态物体
- 还要把运动物体作为对象来建模

### 9.4 Kimera / Kimera-Semantics

`Kimera` 官方页把自己描述为 real-time metric-semantic SLAM 库，使用 camera images 和 inertial data 构建 semantically annotated 3D mesh  
来源：<https://github.com/MIT-SPARK/Kimera>

`Kimera-Semantics` 则是它的语义重建模块  
来源：<https://github.com/MIT-SPARK/Kimera-Semantics>

这条路线很适合理解：

- 语义 SLAM 不只是给点云染色
- 它可以输出可用的 3D 语义 mesh

## 10. 动态处理里最容易踩的坑

### 10.1 把“可动”误当“在动”

比如：

- 人通常可动，但可能站着不动
- 停车场的车通常可动，但此刻可能静止

如果只按语义类别硬剔除，会损失很多可用静态信息。

### 10.2 把“短时静止”误当“长期静态”

反过来，一个人站住几帧也不代表它属于长期地图。

所以动态 SLAM 难就难在：

- 动态不是单帧属性，而是时间属性

### 10.3 只清前端，不清地图

有些系统前端暂时过滤了动态点，但历史地图里已经混入：

- 行人残影
- 物体拖影
- 错误回环候选

所以真正稳的动态 SLAM，往往还要考虑：

- 地图更新策略
- 地图老化与清理
- 静态背景维护

## 11. 语义 SLAM 里最容易踩的坑

### 11.1 语义网络很强，不等于 SLAM 就会更强

因为语义引入后还要面对：

- 标签噪声
- 类别边界误差
- 语义和几何不一致
- 时序稳定性

### 11.2 语义标签怎么融合并不简单

常见问题：

- 单帧标签抖动
- 类别概率累积偏置
- 视角变化导致的分类不稳定

### 11.3 语义信息不一定直接提升轨迹精度

语义 SLAM 的价值很多时候更多体现在：

- 地图更可用
- 任务更高级

而不一定总是体现在：

- ATE 更小

## 12. 数据集怎么选

如果要看动态 / 语义方向，优先关注：

- `TUM RGB-D` 中动态序列
- `KITTI-360`
- `ScanNet`
- `Replica`
- `uHumans / uHumans2`

其中 `uHumans` 是 MIT SPARK 用于测试 Kimera 一类 metric-semantic SLAM 的照片级仿真数据，官方页面明确提供 human 和 object 的 ground truth  
来源：<https://web.mit.edu/sparklab/datasets/uHumans/>

## 13. 项目落地时怎么选路线

### 13.1 如果目标是“先稳住定位”

优先考虑：

- 几何动态剔除
- 轻量语义先验
- 不要一开始就把系统做得过重

### 13.2 如果目标是“机器人真正理解场景”

优先考虑：

- 语义体素 / 语义 mesh
- 实例级建图
- 物体级地图

### 13.3 如果目标是“在动态场景中长期运行”

优先考虑：

- 静态背景地图
- 动态对象过滤与跟踪
- 回环时的动态一致性判断

## 14. 一个很实用的对比表

| 方向 | 核心目标 | 常见输入 | 常见输出 | 主要挑战 |
| --- | --- | --- | --- | --- |
| 动态 SLAM | 动态场景下稳定定位建图 | RGB / RGB-D / stereo / LiDAR | 静态地图、相机轨迹，有时带动态目标轨迹 | 动静分离、误剔除、地图污染 |
| 语义 SLAM | 地图带语义理解能力 | 图像语义分割、实例分割、点云标签 | 语义点云、语义体素、语义 mesh | 标签噪声、时序一致性、计算开销 |
| 动态+语义 SLAM | 同时抗动态并理解场景 | 几何 + 语义联合输入 | 静态背景地图 + 语义对象地图 | 系统复杂、模块耦合强 |

## 15. 面试里怎么答

### 15.1 动态 SLAM 的思路是什么

一个比较顺的答法：

- 传统 SLAM 假设场景静态
- 动态 SLAM 会先检测独立运动区域
- 再剔除或单独建模这些区域
- 最后尽量只用静态背景做定位建图

### 15.2 动态 SLAM 和基础矩阵分割有什么区别

可以这样答：

- 基础矩阵 / 多视图几何分割更偏纯几何动态检测
- 动态 SLAM 是更完整的系统问题，除了检测，还要考虑跟踪、建图、回环、地图维护

### 15.3 语义 SLAM 的优势和劣势

优势：

- 地图更可解释
- 更适合高级任务
- 便于交互、规划和操作

劣势：

- 依赖语义模型
- 计算开销更大
- 标签噪声会传进地图

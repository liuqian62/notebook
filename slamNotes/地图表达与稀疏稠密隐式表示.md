# 地图表达与稀疏稠密隐式表示

这页的目标是把一个经常被割裂的问题串起来：

SLAM 到底在“建什么地图”？

很多时候我们只盯着定位精度，但真正决定系统能力边界的，往往是地图表达本身。

因为不同地图表达会直接影响：

- 能否实时
- 能否做回环和重定位
- 能否做避障和规划
- 能否做高质量重建和渲染
- 能否接语义、编辑和数字孪生

## 1. 先把地图表达分成几类

可以先把常见地图粗分成五类：

### 1.1 稀疏地图

典型表示：

- 路标点
- 关键帧
- 特征描述子

代表系统：

- ORB-SLAM 系列
- 许多传统视觉 SLAM / VIO

### 1.2 半稠密地图

典型表示：

- 仅在高梯度区域恢复深度
- 稀疏点以外补一部分有效像素

代表思路：

- LSD-SLAM 一类方法

### 1.3 稠密几何地图

典型表示：

- 深度图
- 点云
- surfel
- TSDF / ESDF
- mesh

代表系统：

- KinectFusion
- ElasticFusion
- Voxblox

### 1.4 神经隐式地图

典型表示：

- NeRF
- SDF
- occupancy field
- feature grid + MLP

代表系统：

- NICE-SLAM
- Co-SLAM
- Point-SLAM 的一部分路线也和这类表示强相关

### 1.5 3D Gaussian Splatting 地图

典型表示：

- 一组 3D Gaussians
- 每个 Gaussian 带位置、协方差、颜色、透明度等属性

代表方向：

- Gaussian Splatting SLAM
- GS-SLAM

## 2. 为什么“地图表达”会决定系统能力

一个很实用的理解方式是：

- 稀疏地图更像“定位地图”
- 稠密地图更像“几何地图”
- 神经隐式 / 3DGS 更像“可渲染地图”

当然这不是绝对边界，但很适合先建立直觉。

### 2.1 如果主要目标是定位

更常见的是：

- 稀疏特征点
- 关键帧图
- 局部地图

因为它们：

- 计算便宜
- 匹配方便
- 适合回环和重定位

### 2.2 如果主要目标是重建

更常见的是：

- 稠密点云
- surfel
- TSDF / mesh

因为它们：

- 几何更完整
- 更适合表面恢复
- 便于和规划或建模任务结合

### 2.3 如果主要目标是渲染和数字孪生

更常见的是：

- NeRF
- SDF
- 3DGS

因为它们：

- 能表示外观
- 新视角合成能力更强
- 地图不只是“几何”，而是“几何 + 外观”

## 3. 稀疏地图到底是什么

在经典视觉 SLAM 里，地图通常不是“每个位置都建出来”，而是只保留对定位最有用的信息：

- 关键帧
- 稀疏三维点
- 描述子
- 关键帧之间的共视关系

### 3.1 稀疏地图的优点

- 实时性好
- 存储小
- BA 和图优化更容易做
- 回环和重定位成熟

### 3.2 稀疏地图的局限

- 不适合高质量重建
- 对规划和避障支持有限
- 对场景表面表达不完整

所以它更像：

- 为定位服务的工作地图

## 4. 半稠密和稠密地图为什么出现

因为只靠稀疏点，很多任务做不了，比如：

- AR 遮挡
- 机械臂交互
- 重建房间表面
- 做碰撞检测
- 做高质量可视化

这就推动了地图从“少量路标点”走向“更多表面信息”。

## 5. 点云、surfel、mesh 各自代表什么

### 5.1 点云

最直接：

- 一堆三维点

优点：

- 简单
- 通用
- 来自 LiDAR / RGB-D 很自然

缺点：

- 没有显式拓扑
- 表面连续性不强

### 5.2 Surfel

surfel 可以理解为：

- 带法向、半径、颜色等属性的表面元素

相比普通点云，它更适合表示局部表面。

`ElasticFusion` 官方仓库把自己描述为能够构建 dense globally consistent surfel-based maps 的实时稠密视觉 SLAM 系统  
来源：<https://github.com/mp3guy/ElasticFusion>

### 5.3 Mesh

mesh 更进一步，强调：

- 三角面片
- 显式表面连接关系

优点：

- 表面表达清晰
- 可直接用于可视化、仿真、渲染

缺点：

- 在线维护和更新更复杂

## 6. TSDF / ESDF 为什么在机器人里很重要

### 6.1 TSDF

TSDF 是 Truncated Signed Distance Field。

它的核心不是“存点”，而是：

- 在体素里存到最近表面的截断有符号距离

这类表示很适合：

- 融合多帧深度
- 重建连续表面
- 从体素中提取 mesh

`KinectFusion` 是这条路线最经典的代表之一。微软研究院页面描述它会把 Kinect 深度流融合到一个 single global implicit surface model 中，并实时跟踪相机位姿  
来源：<https://www.microsoft.com/en-us/research/publication/kinectfusion-real-time-dense-surface-mapping-tracking/>

### 6.2 ESDF

ESDF 是 Euclidean Signed Distance Field。

和 TSDF 相比，ESDF 更常用于：

- 路径规划
- 避障
- 距障查询

`Voxblox` 官方仓库把自己描述为主要关注 TSDF 和 ESDF 的 flexible voxel-based mapping library，并明确提到能直接从 TSDF 构建 ESDF  
来源：<https://github.com/ethz-asl/voxblox>

所以可以粗略记成：

- `TSDF` 更偏重建
- `ESDF` 更偏规划

## 7. 为什么神经隐式表示会进入 SLAM

传统稠密地图虽然已经能表达几何，但对下面这些能力支持有限：

- 逼真的视角合成
- 高质量外观建模
- 更统一地表示 geometry + appearance

于是 NeRF / SDF 这类神经隐式表示被引入了 SLAM。

### 7.1 神经隐式表示在表达什么

常见会学习：

- 空间某点的颜色
- 密度 / 占据概率
- SDF 值
- 多尺度特征

和传统点云或 TSDF 相比，它不是直接存表面，而是：

- 学一个连续场

### 7.2 它的优势

- 可微
- 连续
- 外观建模能力强
- 新视角合成效果好

### 7.3 它的代价

- 训练 / 优化成本高
- 实时性压力大
- 通常更吃 GPU
- 工程链路更复杂

## 8. NICE-SLAM、Point-SLAM 这类方法在地图上做了什么

`NICE-SLAM` 官方页面说明它是 dense SLAM system，并通过 hierarchical scene representation 来提升大场景室内重建和跟踪  
来源：<https://www.microsoft.com/en-us/research/publication/nice-slam-neural-implicit-scalable-encoding-for-slam/>

`Point-SLAM` 的论文摘要说明它使用 dense neural point cloud-based representation，并在 tracking、mapping 和 rendering accuracy 上对比现有 dense neural SLAM 方法取得了竞争力  
来源：<https://huggingface.co/papers/2304.04278>

这里有个很重要的观察：

- 新一代 SLAM 方法已经不只在争“轨迹误差”
- 还在争“地图能不能同时服务重建、渲染和跟踪”

## 9. 3DGS 为什么这两年这么受关注

`3D Gaussian Splatting` 官方实现仓库把它定义为 “3D Gaussian Splatting for Real-Time Radiance Field Rendering” 的原始参考实现  
来源：<https://github.com/graphdeco-inria/gaussian-splatting>

3DGS 被关注的原因很直接：

- 渲染速度快
- 视觉质量高
- 比很多 NeRF 方法更容易达到实时或近实时显示

这使它很适合用来做：

- 可渲染地图
- 数字孪生
- 交互式浏览

## 10. 3DGS 放进 SLAM 后意味着什么

一旦把 3DGS 放进 SLAM，地图就不再只是“给定位器看的”，而是同时服务：

- 跟踪
- 建图
- 渲染
- 视觉重建评测

这也是为什么 `Gaussian Splatting SLAM`、`GS-SLAM` 一类工作开始出现。

你可以把它理解成：

- 经典 SLAM 主要维护几何工作地图
- 新一代 radiance-field / 3DGS-SLAM 开始维护“可渲染世界模型”

## 11. 稀疏、稠密、隐式、3DGS 之间不是替代关系

这点很重要。

很多人会误以为：

- 有了 NeRF / 3DGS，传统地图就过时了

其实更现实的情况是：

- 不同地图表示服务不同任务

### 11.1 稀疏地图更像前端工作层

适合：

- 快速跟踪
- 回环检索
- 图优化

### 11.2 稠密几何地图更像环境结构层

适合：

- 表面重建
- 体素地图
- 避障和规划

### 11.3 隐式 / 3DGS 地图更像渲染与高保真表达层

适合：

- 视觉重建
- 可渲染地图
- 数字孪生

未来很可能不是“谁替代谁”，而是：

- 多层地图并存

## 12. 如果从机器人任务角度选地图表达

### 12.1 只想稳定定位

优先考虑：

- 稀疏地图
- 关键帧图
- 局部点云地图

### 12.2 想做重建 + 交互

优先考虑：

- surfel
- TSDF
- mesh

### 12.3 想做规划

优先考虑：

- occupancy
- TSDF / ESDF
- 可快速距离查询的体素地图

### 12.4 想做渲染和数字孪生

优先考虑：

- NeRF
- SDF
- 3DGS

## 13. 如果从 SLAM 论文阅读角度看地图表达

以后看论文时，可以主动问这几个问题：

1. 地图是怎么表示的  
2. 地图是为定位服务，还是为重建 / 渲染服务  
3. 地图更新是离散的还是连续的  
4. 地图能否直接用于回环、规划、渲染  
5. 地图代价是算力瓶颈，还是存储瓶颈  

这样你会比只看 ATE / RPE 更快抓住论文本质。

## 14. 一个很实用的对比表

| 地图类型 | 典型表示 | 主要优势 | 主要局限 | 更适合的任务 |
| --- | --- | --- | --- | --- |
| 稀疏地图 | 路标点、关键帧 | 快、轻、适合回环与 BA | 表面不完整 | 定位、回环、重定位 |
| 半稠密地图 | 高梯度区域深度 | 比稀疏更丰富 | 仍不完整 | 轻量重建、视觉跟踪 |
| 点云 / surfel | 点或表面元 | 直观、几何丰富 | 拓扑弱或维护复杂 | 稠密建图、AR、可视化 |
| TSDF / ESDF | 体素距离场 | 重建与规划友好 | 内存和更新代价较高 | 重建、避障、路径规划 |
| 神经隐式 | 连续场 | 表达连续、外观强 | 优化开销大 | 可渲染建图、重建 |
| 3DGS | 3D 高斯集合 | 渲染快、质量高 | 工程链条仍在发展 | 数字孪生、可渲染 SLAM |

## 15. 面试里怎么答“地图表达有什么区别”

一个比较顺的答法是：

1. 先按信息密度分  
   有稀疏、半稠密、稠密

2. 再按表示方式分  
   有点、surfel、体素、mesh、隐式场、3D 高斯

3. 再按任务分  
   定位更偏稀疏，规划更偏体素距离场，渲染更偏隐式和 3DGS

4. 最后补一句趋势  
   现代 SLAM 越来越不是只维护一个单层地图，而是可能同时维护定位地图、几何地图和可渲染地图

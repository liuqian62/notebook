# SLAM 回环检测

这页希望把一个问题讲清楚：

为什么 SLAM 明明前端已经在估计位姿了，还是需要回环检测？

最短答案是：

- 前端里程计只能尽量减小局部误差
- 但误差会随时间积累
- 回环检测负责发现“我回到了以前来过的地方”
- 一旦确认是回环，就能把长时间积累的漂移通过图优化拉回去

## 1. 回环检测在 SLAM 里到底做什么

一个完整 SLAM 系统一般可以粗略分成：

- 前端：连续估计相邻时刻运动
- 地图管理：维护关键帧、路标、局部地图
- 回环检测：判断当前观测是否对应历史地点
- 后端优化：利用回环约束做全局一致性优化

所以回环检测本质上不是“优化器”，而是“生成全局约束”的模块。

它解决的核心不是短时跟踪，而是：

- 累计漂移
- 长时间运行后的全局一致性
- 重定位 / revisiting 场景下的地图闭合

## 2. 一个标准回环模块通常怎么做

大多数回环系统都可以拆成四步：

### 2.1 候选地点检索

先从历史关键帧或历史子地图里，检索出“可能是同一个地方”的候选。

常见方法：

- 视觉 BoW / binary BoW
- 全局图像描述子
- 序列匹配
- LiDAR 全局描述子

这一步追求的是：

- 快
- 召回率高

但通常允许一定误检，因为后面还会做几何验证。

### 2.2 几何一致性验证

拿到候选之后，要确认它不是“看起来像”，而是真的在几何上对得上。

视觉里常见：

- 特征匹配
- RANSAC
- Essential / Fundamental / PnP / Sim(3) 验证

激光里常见：

- Scan matching
- ICP / GICP / NDT
- 位姿先验 + 局部地图配准

这一步是在做“去假阳性”。

### 2.3 构造回环约束

如果验证通过，就会得到一个相对位姿约束，比如：

- `T_ij`
- `SE(3)` 约束
- 单目场景里可能需要 `Sim(3)` 约束

为什么单目常见 `Sim(3)`：

- 单目系统容易产生尺度漂移
- 光有刚体位姿约束不一定足够纠正尺度问题

### 2.4 图优化或位姿图校正

最后把回环边加到图里，执行：

- pose graph optimization
- essential graph optimization
- global BA

这一步才是真正把全局轨迹“拉直”的过程。

## 3. 为什么回环难做

回环检测并不是简单的图像检索，它至少同时面对四类问题。

### 3.1 外观变化

同一个地方在不同时间可能有：

- 光照变化
- 曝光变化
- 视角变化
- 遮挡
- 动态物体

### 3.2 场景混淆

不同位置可能“长得很像”，比如：

- 长走廊
- 地下车库
- 办公室工位区
- 林地 / 隧道 / 管廊

这会带来 perceptual aliasing，也就是感知混淆。

### 3.3 重复纹理

重复窗户、地砖、货架、栅栏、机柜，都会让局部特征匹配看起来很多，但其中大量是错误对应。

这也是为什么你的仓库里之前会专门关注“低纹理和重复纹理回环”。

### 3.4 实时性

回环通常要在越来越大的历史地图里找候选，所以必须兼顾：

- 检索效率
- 内存开销
- 在线更新

## 4. 视觉回环的主流路线

### 4.1 Bag of Words

这是最经典的一条路线。

基本思想：

1. 提取局部特征
2. 把特征量化成“视觉词”
3. 用词频向量表示图像
4. 用倒排索引做快速检索

优点：

- 工程成熟
- 检索快
- 很适合关键帧级 place recognition

缺点：

- 对强视角变化、强光照变化、动态遮挡有局限
- 容易受 burstiness 和 perceptual aliasing 影响

`DBoW2` 是这一类最经典的开源库之一，官方仓库把它描述为用于 image retrieval 和 loop closing 的 bag-of-words library  
来源：<https://github.com/dorian3d/DBoW2>

### 4.2 Binary BoW / 增量词袋

当系统使用 ORB、BRIEF 这类二进制特征时，binary BoW 会更合适。

`iBoW-LCD` 是一个很典型的例子，它强调 incremental bag of binary words，用于 appearance-based loop closure detection  
来源：<https://github.com/emiliofidalgo/ibow-lcd>

这类方法的优点是：

- 更适合 ORB-SLAM 风格前端
- 在线词典更新更自然
- 工程实现相对直接

### 4.3 基于 hashing 的检索

例如 `MILD`，核心思路是利用 multi-index hashing 来加速二进制特征检索。

适合：

- 二进制描述子
- 大规模在线回环库
- 对检索速度比较敏感的场景

`MILD` 仓库直接对应论文《Multi-Index Hashing for Loop closure Detection》  
来源：<https://github.com/lhanaf/MILD>

### 4.4 学习式 place recognition

近几年越来越多方法不再只依赖手工词袋，而是学习一个全局场景描述子。

常见优点：

- 对视角和外观变化更强
- 检索阶段召回率更高

常见问题：

- 训练数据依赖更强
- 迁移到新场景不一定稳定
- 通常仍需要几何验证，否则误检依旧危险

所以工程里常见的做法不是“学习描述子替代一切”，而是：

- 学习式检索做候选召回
- 几何验证做最后确认

### 4.5 序列匹配

如果单帧容易混淆，可以利用连续帧的时间一致性。

序列方法常见优点：

- 对长走廊、重复结构更稳
- 单帧误检可以被时间约束抑制

代价是：

- 延迟更高
- 对运动模式变化更敏感

## 5. 激光回环的主流路线

激光回环和视觉回环的思想相同，但表示方式不同。

### 5.1 Scan Context

`Scan Context` 是 LiDAR 回环里非常经典的一条路线。

核心想法是把点云扫描投影成环形 / 扇区结构描述子，再做快速检索。

它的优势是：

- 对旋转有较好的处理方式
- 检索效率高
- 很适合大规模 LiDAR place recognition

官方仓库将其描述为 egocentric spatial descriptor for place recognition within 3D point cloud map  
来源：<https://github.com/gisbi-kim/scancontext_tro>

### 5.2 激光检索 + ICP / NDT 验证

很多实际系统会先：

- 用全局描述子找候选

再：

- 用 ICP / GICP / NDT 做精配准

这样组合的原因很直接：

- 全局描述子负责快检索
- 精配准负责低误检

### 5.3 在 LIO 系统里的回环

LIO / LOAM 类系统里，回环通常还要考虑：

- 运动畸变
- 地图尺度大
- 环境重复结构
- 优化器中 IMU / LiDAR 因子的耦合关系

所以一个好的 LiDAR 回环模块，不只是“找到像的帧”，还要考虑：

- 约束质量够不够好
- 加进图里会不会把系统拉坏

## 6. 视觉和激光回环有什么核心区别

可以用一句话概括：

- 视觉更容易受外观变化影响
- 激光更容易受结构重复和场景退化影响

再细一点看：

### 6.1 视觉回环

更依赖：

- 外观相似性
- 图像纹理
- 特征稳定性

更怕：

- 光照变化
- 动态遮挡
- 重复纹理

### 6.2 激光回环

更依赖：

- 几何结构
- 空间布局
- 点云覆盖率

更怕：

- 长直走廊
- 稀疏点云
- 场景几何重复
- 大范围空旷区域

## 7. 回环之后到底优化什么

很多人容易把“检测到回环”和“系统误差已经修正”混为一谈，其实它们不是一件事。

检测到回环之后，常见还有三层工作：

### 7.1 位姿图优化

把关键帧看成节点，里程计约束和回环约束看成边，做全局一致性优化。

优点：

- 快
- 工程上常用
- 容易在线运行

### 7.2 Essential Graph / 稀疏图优化

以 ORB-SLAM 系统为代表，会维护一张更稀疏但足够表达全局结构的图来加速优化。

适合：

- 大图
- 需要快速闭环校正

### 7.3 Global BA

这是更重的一步，通常精度更高，但开销也更大。

适合：

- 离线精修
- 闭环后后台慢慢优化

## 8. 工程里怎么降低误检风险

这部分很重要，因为回环模块最怕的不是“漏检”，而是“错检后把图拉坏”。

常见做法：

- 设置时间窗口，排除最近邻关键帧，避免把局部连续帧误当回环
- 至少要求若干连续帧都支持同一候选
- 候选检索后必须做几何验证
- 对估计出的相对位姿设置信息矩阵或置信度门限
- 对大残差回环边使用 robust kernel
- 回环后先局部验证，再触发全局优化

经验上：

- 漏掉一两个回环通常只是精度差一点
- 加入一个错误回环，整张图可能直接变形

## 9. 低纹理和重复纹理场景该怎么想

这是你现有笔记里已经提过的重点，这里把主线收束一下。

### 9.1 低纹理

问题：

- 特征点少
- 描述子区分度差
- 候选召回和几何验证都变难

可考虑：

- 引入 IMU / LiDAR
- 用更稳的局部特征或全局描述子
- 借助序列信息

### 9.2 重复纹理

问题：

- 匹配点很多，但真假混杂
- 视觉词频会被 burstiness 放大
- 容易出现高分错误候选

可考虑：

- burstiness 抑制
- 更强的几何一致性校验
- 序列级约束
- 多传感器辅助验证

## 10. 面试里最常被问的几个点

### 10.1 为什么需要回环

标准答法：

- 因为里程计误差会累积
- 回环用于发现历史重访
- 回环约束进入图优化后可抑制累计漂移

### 10.2 回环检测和重定位有什么区别

可以这样区分：

- 回环检测：系统已经在跟踪，只是判断当前地点是否曾来过
- 重定位：系统跟踪丢失后，重新找回自己在地图中的位姿

两者经常共享 place recognition 模块，但触发场景不同。

### 10.3 为什么回环后能修正尺度漂移

针对单目：

- 单目轨迹会有尺度漂移
- 回环时如果估计 `Sim(3)` 约束
- 再做图优化，就能把历史轨迹尺度拉回一致

### 10.4 回环最难的点是什么

一个比较完整的回答是：

- 候选召回率和误检率要平衡
- 外观变化和感知混淆难处理
- 错误回环的破坏性远大于漏检
- 大规模场景里还要兼顾实时性

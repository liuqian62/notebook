# 基础四大件

这个目录集中整理求职、复习和面试常见的计算机基础知识，内容以“总览笔记 + 高频面试点”为主，适合作为一套统一入口来回顾。

## 目录导航

- [数据结构](./数据结构.md)：数组、链表、栈、队列、树、图、哈希、堆、并查集
- [算法](./算法.md)：复杂度、查找、排序、递归、分治、贪心、动态规划、回溯
- [计组](./计组.md)：CPU、指令执行、存储层次、Cache、总线、DMA、中断
- [OS](./OS.md)：进程线程、内存管理、调度、同步、死锁、文件系统、I/O
- [计网](./计网.md)：分层模型、TCP/UDP、HTTP/HTTPS、握手挥手、网络排错
- [mysql](./mysql.md)：SQL、索引、事务、锁、日志、优化、主从复制
- [MySQL 专题索引](./mysql专题索引.md)：按基础、进阶、运维重新组织入口
- [mysql/readme.md](./mysql/readme.md)：目录化专题入口，便于后续继续拆分
- [设计模式](./设计模式.md)：六大原则、创建型、结构型、行为型模式
- [计算机基础专题索引](./计算机基础专题索引.md)：速查入口和外部补充资料

## 推荐阅读顺序

1. 想系统复习基础：按 `数据结构 -> 算法 -> OS -> 计网 -> 计组`
2. 想面试前速查：先看 [计算机基础专题索引](./计算机基础专题索引.md)，再按薄弱项回到对应主题
3. 想补工程面和后端面：先看 [MySQL 专题索引](./mysql专题索引.md)，再看 [mysql](./mysql.md) 和 [设计模式](./设计模式.md)

## 使用建议

- 第一轮阅读先抓“概念边界”和“为什么这样设计”
- 第二轮重点记高频对比题，例如数组和链表、进程和线程、TCP 和 UDP
- 第三轮用自己的话复述每个主题，确认能在 1 到 3 分钟内讲清楚

## 目录说明

- 主体文档统一采用“总览 + 高频点 + 复习建议”的结构
- 这份 `readme.md` 主要负责导航，不堆过多细节
- 如果某个主题后续内容继续膨胀，再考虑拆成子目录或专题文档

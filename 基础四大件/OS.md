## 操作系统在做什么

操作系统本质上是硬件和应用程序之间的一层管理软件，核心目标有三件事：

- 管理资源：CPU、内存、磁盘、网络、外设
- 提供抽象：进程、线程、文件、虚拟内存、套接字
- 保证隔离与共享：让多个程序安全、高效地一起运行

站在用户态程序的角度，操作系统主要提供：

- 运行程序的环境
- 访问硬件的统一接口
- 权限控制和安全边界
- 并发执行和资源分配能力

## 进程与线程

### 进程

进程是资源分配的基本单位，可以理解为“一个正在运行的程序实例”。

一个进程通常拥有：

- 独立的虚拟地址空间
- 代码段、数据段、堆、栈
- 打开的文件描述符
- 各类内核资源和运行状态

进程的典型状态：

- 就绪态：已经具备运行条件，等待 CPU
- 运行态：正在 CPU 上执行
- 阻塞态：等待 I/O、锁、事件等资源

### 线程

线程是 CPU 调度的基本单位，是进程中的执行流。

同一进程中的线程通常共享：

- 地址空间
- 全局变量和堆
- 打开的文件

线程私有：

- 程序计数器
- 寄存器上下文
- 栈

### 进程和线程的区别

- 进程更重，线程更轻
- 进程切换开销通常更大，因为涉及地址空间等资源切换
- 线程切换开销较小，但线程间共享内存，也更容易出现同步问题
- 进程隔离性更好，线程通信更方便

### 用户线程与内核线程

- 用户线程由用户态库管理，切换快，但一个线程阻塞可能影响整个进程
- 内核线程由内核调度，能更好利用多核，也更适合阻塞型任务

### 上下文切换

上下文切换是指 CPU 从一个执行流切换到另一个执行流时，保存和恢复运行现场。

切换成本主要来自：

- 寄存器保存与恢复
- 内核态切换
- cache 命中率下降
- 进程切换时可能涉及页表切换、TLB 刷新

## 进程通信 IPC

常见 IPC 方式：

- 管道 `pipe`
- 命名管道 `FIFO`
- 消息队列
- 共享内存
- 信号量
- 信号
- 套接字 `socket`

### 常见方式对比

- 管道：简单，适合有亲缘关系的进程，通常是半双工
- 消息队列：以消息为单位，方便解耦，但有系统开销
- 共享内存：最快，因为数据不需要多次拷贝，但必须自己处理同步
- 套接字：最通用，可用于同机或跨网络通信
- 信号：适合通知，不适合携带复杂数据

### 为什么共享内存最快

因为多个进程直接映射到同一块物理内存，减少了用户态和内核态之间的数据拷贝。但“快”只代表传输快，不代表更好用，因为同步复杂度更高。

## 线程同步与互斥

多个线程共享数据时，必须解决竞态条件问题。

### 互斥与同步

- 互斥：某一时刻只允许一个线程访问临界区
- 同步：控制线程执行顺序，保证先后依赖关系

### 常见同步机制

- 互斥锁 `mutex`
- 读写锁
- 自旋锁
- 信号量 `semaphore`
- 条件变量 `condition variable`

### 互斥锁和自旋锁

- 互斥锁拿不到锁时通常会阻塞并让出 CPU，适合临界区较长的情况
- 自旋锁拿不到锁时会循环等待，适合临界区极短、线程不会被长时间挂起的情况

### 信号量和互斥锁的区别

- 互斥锁更强调“独占访问”
- 信号量更像“资源计数器”，可以控制同时访问资源的线程数量

### 条件变量

条件变量通常配合互斥锁使用，用来解决“条件不满足就等待，满足后再唤醒”的问题，典型场景是生产者消费者模型。

### 生产者消费者问题

核心点：

- 互斥访问缓冲区
- 缓冲区满时生产者等待
- 缓冲区空时消费者等待

常见实现工具：

- 一个互斥锁
- 两个条件变量或两个信号量

## 调度

调度的目标通常包括：

- 提高 CPU 利用率
- 提高吞吐量
- 降低响应时间
- 保证公平性

### 常见调度算法

#### 先来先服务 FCFS

- 按到达顺序执行
- 实现简单
- 对长作业友好，对短作业不友好

#### 短作业优先 SJF

- 优先执行预计运行时间短的任务
- 平均等待时间通常较优
- 可能导致长作业饥饿

#### 时间片轮转 RR

- 每个任务分配固定时间片
- 适合分时系统
- 时间片太大退化为 FCFS，太小则切换开销大

#### 优先级调度

- 优先运行优先级高的任务
- 可能产生低优先级任务饥饿

#### 多级反馈队列

面试中非常高频，核心思想是：

- 不同优先级队列使用不同时间片
- 新任务先进入高优先级队列
- CPU 密集型任务逐渐下沉
- I/O 密集型任务通常更容易保留高优先级

这种方式兼顾了响应速度和吞吐量。

## 死锁

### 死锁的四个必要条件

- 互斥
- 请求与保持
- 不可剥夺
- 循环等待

### 处理死锁的思路

- 预防：破坏四个必要条件之一
- 避免：在资源分配时避免进入不安全状态，例如银行家算法
- 检测与解除：允许死锁发生，之后检测并恢复

### 常见避免手段

- 按固定顺序申请锁，破坏循环等待
- 一次性申请所需资源，破坏请求与保持
- 使用超时机制、死锁检测工具

## 内存管理

内存管理的目标：

- 提高内存利用率
- 支持并发执行
- 保证进程隔离
- 给程序提供统一的地址空间抽象

### 地址空间

程序中访问的是虚拟地址，不是直接访问物理地址。

地址翻译过程通常是：

- CPU 生成虚拟地址
- MMU 根据页表完成虚拟地址到物理地址的映射
- 如有需要借助 TLB 加速

### 堆和栈

- 栈：由编译器和运行时自动管理，主要存放局部变量、函数调用信息
- 堆：由程序主动申请和释放，适合动态分配的大块或生命周期不固定的数据

一般特点：

- 栈分配释放快，但空间较小
- 堆更灵活，但容易产生碎片和内存泄漏

### 内存碎片

- 内部碎片：分配出去但没完全用上的空间
- 外部碎片：空闲空间总量足够，但不连续，导致无法分配大块内存

分页主要解决外部碎片问题，分段更符合逻辑结构但管理更复杂。

## 虚拟内存

虚拟内存让程序“感觉自己拥有一整块连续内存”，其核心收益有：

- 扩大可用地址空间
- 提高内存利用率
- 支持按需加载
- 增强进程隔离

### 分页

分页把虚拟内存和物理内存都划分为固定大小的页。

优点：

- 管理方便
- 没有外部碎片
- 易于实现离散分配

缺点：

- 会有页表开销
- 可能存在内部碎片

### 页面置换算法

常见页面置换算法：

- OPT：理论最优，实际无法准确实现
- FIFO：实现简单，但可能出现 Belady 异常
- LRU：更符合局部性原理，实践中常用近似实现
- CLOCK：对 LRU 的工程化近似

### 缺页中断

当访问的页面不在内存中时，会触发缺页中断。操作系统会：

- 判断访问是否合法
- 从磁盘把对应页面调入内存
- 必要时淘汰旧页面
- 更新页表后继续执行

### 快表 TLB

TLB 是页表项缓存，用来加速地址转换。

如果没有 TLB，CPU 每次访存都要先查页表，成本很高；有了 TLB，多数情况下可以直接命中缓存得到物理页号。

## 文件系统

文件系统负责管理持久化数据，提供“文件”和“目录”的抽象。

### 文件系统做的事情

- 组织文件和目录
- 管理磁盘空间
- 维护元数据
- 提供访问权限控制
- 保证数据一致性

### 文件的常见属性

- 文件名
- 大小
- 创建时间、修改时间
- 权限
- 所有者
- 数据块位置

### inode

在类 Unix 系统中，`inode` 记录文件元数据，一般不直接存储文件名。目录本质上维护的是“文件名 -> inode”的映射关系。

所以：

- 同一个文件可以有多个硬链接
- 删除文件名不一定立刻删除文件数据
- 当链接计数为 0 且没有进程占用时，文件数据才真正可回收

### 硬链接和软链接

- 硬链接：多个文件名指向同一个 inode
- 软链接：相当于保存目标路径的特殊文件

区别：

- 硬链接通常不能跨文件系统
- 软链接可以跨文件系统，也可以指向目录
- 删除源文件后，硬链接通常仍可访问，软链接会失效

### 磁盘调度

常见磁盘调度算法：

- FCFS
- SSTF
- SCAN
- C-SCAN

其中 SCAN 常被称为“电梯算法”。

## I/O 模型

I/O 的两个关键阶段：

- 等待数据准备好
- 把数据从内核拷贝到用户空间

### 阻塞 I/O

调用线程一直等，直到数据准备好并拷贝完成。

### 非阻塞 I/O

如果数据没准备好，立刻返回，调用方不断轮询。

### I/O 多路复用

典型接口：

- `select`
- `poll`
- `epoll`

核心思想是一个线程同时监听多个文件描述符，当某个描述符就绪时再处理。

`select`、`poll`、`epoll` 的常见区别：

- `select` 有连接数上限，且每次都要拷贝和遍历整个集合
- `poll` 去掉了固定上限，但本质仍需轮询
- `epoll` 更适合高并发场景，就绪通知更高效

### 同步 I/O 和异步 I/O

- 同步 I/O：数据拷贝阶段通常仍由调用线程参与
- 异步 I/O：内核把整个 I/O 做完后再通知应用

## 中断与系统调用

### 中断

中断是硬件或软件打断当前 CPU 执行流程的一种机制，用来处理外设事件、时钟事件等。

### 系统调用

系统调用是用户程序主动请求内核提供服务的入口，例如：

- 创建进程
- 申请内存
- 读写文件
- 网络通信

用户态程序不能直接执行特权操作，所以需要通过系统调用陷入内核态。

### 用户态和内核态

- 用户态权限低，应用程序运行在这里
- 内核态权限高，可以访问硬件和执行特权指令

切换到内核态的常见场景：

- 系统调用
- 中断
- 异常

## 常见面试题速答

### 进程和线程的区别

- 进程是资源分配的基本单位，线程是调度的基本单位
- 进程有独立地址空间，线程共享进程地址空间
- 线程切换开销更小，但同步更复杂

### 协程和线程的区别

- 协程通常是用户态的轻量级执行单元
- 线程由操作系统调度，协程由运行时或程序自己调度
- 协程切换更轻，但不能完全替代多核并行

### 为什么要有虚拟内存

- 为进程提供独立地址空间
- 支持按需加载和换页
- 提高内存利用率
- 让程序编写和运行更简单

### 为什么页表不能做得特别大

页表本身要占内存，而且进程越多开销越大，所以实际系统常用多级页表、反向页表、TLB 等手段降低成本。

### 为什么线程崩溃往往会导致整个进程崩溃

因为线程共享进程地址空间。一个线程发生非法内存访问，破坏的是整个进程的内存状态，内核通常会直接终止整个进程。

### `fork` 之后父子进程共享内存吗

逻辑上彼此独立，物理上常通过写时复制 `Copy-On-Write` 提高效率。也就是初始时共享物理页，只有发生写入时才真正复制。

### 僵尸进程和孤儿进程

- 僵尸进程：子进程已经退出，但父进程还没回收其退出状态
- 孤儿进程：父进程先退出，子进程会被其他进程接管，通常是 `init/systemd`

### 互斥锁和读写锁怎么选

- 读多写少时优先考虑读写锁
- 写多或竞争激烈时，读写锁不一定比互斥锁更好

### 怎么避免死锁

- 固定加锁顺序
- 缩小锁粒度
- 避免持锁做耗时操作
- 使用超时、检测和监控机制

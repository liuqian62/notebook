## 计算机组成原理在学什么

这门内容主要回答两个问题：

- 计算机内部是怎么工作的
- 程序最终是如何被硬件执行的

常见关注点有：

- CPU 如何取指、译码、执行
- 数据如何在寄存器、内存、磁盘之间流动
- 为什么 cache 比主存快
- 一条机器指令是怎么访问操作数的

## 冯诺依曼结构

冯诺依曼结构的核心特点：

- 程序和数据都存放在存储器中
- 计算机由运算器、控制器、存储器、输入设备、输出设备构成
- 指令按顺序执行，遇到跳转时改变执行流

它奠定了现代计算机的基本工作方式。

## CPU 基本组成

CPU 通常可以拆成：

- 运算器：负责算术和逻辑运算
- 控制器：负责发出控制信号，协调各部件工作
- 寄存器组：暂存数据、地址和中间结果

常见寄存器：

- 程序计数器 `PC`
- 指令寄存器 `IR`
- 累加器或通用寄存器
- 状态寄存器 `PSW`

## 指令执行流程

一条指令通常经历：

- 取指：根据 `PC` 从主存取出指令
- 译码：分析操作码和操作数信息
- 执行：完成运算、访存或跳转
- 写回：把结果写回寄存器或内存

如果是流水线 CPU，不同阶段还能并行重叠。

### 流水线

流水线的核心目标是提升吞吐量，而不是缩短单条指令延迟。

常见问题：

- 结构相关
- 数据相关
- 控制相关

解决思路包括：

- 增加硬件资源
- 数据前递
- 分支预测

## 存储层次结构

存储系统通常按“速度快但贵”到“速度慢但便宜”分层：

- 寄存器
- Cache
- 主存
- 磁盘

这样设计的核心原因是：

- 兼顾速度、容量和成本
- 利用程序的局部性原理

### 局部性原理

- 时间局部性：刚访问过的数据，短时间内还可能访问
- 空间局部性：访问某个位置后，附近位置也可能很快被访问

## 主存与地址

主存按地址编址，每个存储单元都有唯一地址。

常见概念：

- 字节编址
- 字长
- 按字访问和按字节访问

面试中经常考：

- 32 位和 64 位系统可寻址范围
- 一个地址能表示多大空间

## Cache

Cache 是位于 CPU 和主存之间的高速缓冲。

### 为什么需要 Cache

因为 CPU 速度远高于主存，如果每次都直接访问主存，CPU 会频繁等待。

### Cache 的工作基础

依赖局部性原理，把未来大概率会再次访问的数据提前缓存起来。

### Cache 命中

- 命中：要的数据已经在 Cache 中
- 未命中：需要去主存取数据

### 常见映射方式

- 直接映射
- 全相联映射
- 组相联映射

对比：

- 直接映射简单、快，但冲突可能多
- 全相联最灵活，但硬件成本高
- 组相联是工程里常见折中

### Cache 和虚拟内存的区别

- Cache 解决的是速度差问题
- 虚拟内存解决的是容量、隔离和地址抽象问题

## 指令系统与寻址

### 指令通常包含什么

- 操作码
- 地址码

操作码说明“做什么”，地址码说明“对谁做”。

### 常见寻址方式

- 立即寻址
- 直接寻址
- 间接寻址
- 寄存器寻址
- 寄存器间接寻址
- 基址寻址
- 变址寻址

面试高频点：

- 立即寻址速度快，但灵活性差
- 寄存器寻址快，因为不必访问主存

## 总线与 I/O

总线是连接各部件的数据通路。

常见总线分类：

- 数据总线
- 地址总线
- 控制总线

I/O 设备和主机之间的数据交换方式常见有：

- 程序查询方式
- 中断方式
- DMA 方式

### DMA

DMA 允许外设和内存之间直接传输数据，减少 CPU 参与，适合大块数据传输。

## 中断

中断是外设或系统事件通知 CPU 处理的一种机制。

典型过程：

- 当前程序被打断
- 保存现场
- 转去执行中断服务程序
- 恢复现场
- 返回原程序

中断的价值在于：

- 提高 CPU 利用率
- 让 CPU 不必一直轮询设备状态

## 性能指标

常见性能指标：

- 主频
- CPI
- CPU 执行时间
- 吞吐量
- 响应时间

### 常见公式

CPU 执行时间常写为：

- `CPU 执行时间 = 指令数 × CPI × 时钟周期`

其中：

- 指令数表示程序规模
- CPI 表示平均每条指令需要的时钟周期数
- 时钟周期反映时钟速度

## 常见面试题速答

### CPU 为什么比内存快很多

因为 CPU 使用更高速、更昂贵的电路，而主存更强调容量和成本，两者设计目标不同。

### 为什么要有 Cache

为了缓解 CPU 和主存之间的速度鸿沟，提高访存效率。

### 程序计数器 PC 的作用是什么

保存下一条将要执行的指令地址。

### 流水线的作用是什么

提高指令吞吐量，让多条指令在不同阶段并行执行。

### 中断和异常有什么区别

- 中断通常来自外部事件
- 异常通常来自当前指令执行过程中的错误或特殊情况

### DMA 为什么快

因为它减少了 CPU 在数据搬运上的参与，让数据直接在设备和内存之间传输。

# 贡献与维护说明

这个仓库更偏个人知识库，整理目标是“入口清晰、路径稳定、方便继续补内容”，不是一次性做大规模重构。

## 新内容应该放哪里

- 优先归入现有一级目录，不轻易新增新的同级分类
- 同一主题如果已经有目录，优先补到该目录并更新它的 `readme.md`
- 长内容优先新建独立 Markdown 文件，不要继续把正文堆进目录首页

## README / 索引页怎么写

- 目录级 `readme.md` 主要做导航、阅读顺序和入口说明
- 主题正文尽量写到独立文件
- 如果当前只是占位页，要明确写清“现在有什么、后续准备补什么、先看哪里”

## 提交前自检

1. 检查是否把新内容放进了最合适的目录
2. 检查对应目录的 `readme.md` 是否需要补链接
3. 检查新增图片和资源是否就近存放
4. 运行一次 Markdown 本地链接检查

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\check-markdown-links.ps1
```

## 链接、图片、命名约定

- 仓库内部文档优先使用相对路径链接
- 图片优先放在对应主题目录；公共图片再放公共资源目录
- 文件名尽量语义化，避免只有作者自己能看懂的缩写
- 历史路径尽量保持稳定，避免为了统一风格大规模改名

## 不建议做的整理动作

- 大批量移动文件
- 只为“看起来统一”而重写旧正文
- 不补回链就直接改目录结构
- 在没有必要的情况下新增新的一级主题目录
